# 🛡️ FakeShield — Fake Site Detector

A full-stack web application to detect phishing / fake websites using multi-factor analysis.

## Tech Stack

| Layer      | Technology              | Purpose                                    |
|------------|-------------------------|--------------------------------------------|
| Backend    | Java 17 + Spring Boot 3 | REST API, business logic, scoring engine   |
| Frontend   | HTML / CSS / JavaScript | Single-page app, real-time UI              |
| MySQL      | 8.x via Spring Data JPA | Site reports, blacklist/whitelist, stats   |
| MongoDB    | 6.x via Spring Data     | Detailed per-check analysis logs (flexible schema) |

---

## Architecture

```
Frontend (index.html)
        │
        │  HTTP / JSON
        ▼
Spring Boot (port 8080)
  ├── SiteDetectorController  ← REST endpoints
  ├── SiteAnalysisService     ← scoring engine
  │       ├── URL feature analysis
  │       ├── HTTP probing (SSL, redirects)
  │       ├── Blacklist/whitelist check
  │       └── Brand impersonation detection
  ├── MySQL (JPA)
  │       ├── site_reports    ← summary score per URL
  │       ├── domain_list     ← blacklist / whitelist
  │       ├── user_reports    ← community flags
  │       └── analysis_stats  ← daily aggregates
  └── MongoDB
          └── analysis_logs   ← full check-by-check details
```

## Setup

### Prerequisites
- Java 17+
- Maven 3.8+
- MySQL 8.x
- MongoDB 6.x

### 1. MySQL setup

```sql
CREATE DATABASE fake_detector_db;
-- Then run src/main/resources/schema.sql
mysql -u root -p fake_detector_db < src/main/resources/schema.sql
```

### 2. MongoDB setup

MongoDB needs no schema — Spring Data creates the collection automatically.
Make sure `mongod` is running on `localhost:27017`.

### 3. Configure credentials

Edit `src/main/resources/application.properties`:

```properties
spring.datasource.username=root
spring.datasource.password=YOUR_PASSWORD
spring.data.mongodb.uri=mongodb://localhost:27017/fake_detector_logs
```

### 4. Build & run

```bash
mvn clean package -DskipTests
java -jar target/fake-site-detector-1.0.0.jar
```

The API starts on **http://localhost:8080**

### 5. Open the frontend

Open `frontend/index.html` in any browser (no build step needed).

## REST API Reference

| Method | Endpoint              | Description                       |
|--------|-----------------------|-----------------------------------|
| POST   | /api/v1/analyse       | Analyse a URL                     |
| GET    | /api/v1/report/{id}   | Fetch MySQL report by ID          |
| GET    | /api/v1/log/{mongoId} | Fetch MongoDB detail log          |
| GET    | /api/v1/history       | Recent scans (filter by verdict)  |
| GET    | /api/v1/stats         | Aggregate statistics              |
| POST   | /api/v1/report-fake   | Submit a community fake report    |

### Sample request

```bash
curl -X POST http://localhost:8080/api/v1/analyse \
  -H "Content-Type: application/json" \
  -d '{"url":"https://paypa1-secure-login.tk"}'
```

### Sample response

## Detection checks

| Check                  | Signal                                           | Max Score |
|------------------------|--------------------------------------------------|-----------|
| Blacklist match        | Domain in known-bad list                         | +50       |
| IP in URL              | Raw IP instead of domain name                    | +25       |
| @ symbol in URL        | Hides real destination                           | +20       |
| No HTTPS               | Plain HTTP, no encryption                        | +15       |
| Suspicious TLD         | .tk .ml .xyz .top etc.                           | +15       |
| Brand impersonation    | Brand keyword + domain mismatch                  | +20       |
| Long URL               | > 75 characters                                  | +10       |
| Excessive subdomains   | > 2 subdomain levels                             | +10       |
| Suspicious keywords    | login/verify/secure etc. in URL                  | +10       |
| Double slash           | Obfuscation technique                            | +10       |
| Whitelist match        | Domain in trusted list                           | −20       |

**Verdict thresholds:**
- 0–20 → ✅ SAFE
- 21–50 → ⚠️ SUSPICIOUS
- 51–100 → ❌ FAKE
